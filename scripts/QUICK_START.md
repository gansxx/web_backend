# R2 CLI 快速开始指南

## 🚀 5分钟上手

### 1. 启动脚本

```bash
cd /root/self_code/web_backend
uv run python scripts/r2_cli.py
```

### 2. 常用操作速查

| 操作 | 菜单编号 | 说明 |
|------|----------|------|
| 上传包 | 1 | 上传新的软件包 |
| 查看包信息 | 2 | 查看包的详细信息 |
| 搜索包 | 4 | 按关键词搜索包 |
| 列出公开包 | 5 | 查看所有公开的包 |
| 查看我的上传 | 6 | 查看自己上传的包 |
| 更新包 | 7 | 修改包的描述、标签等 |
| 删除包 | 8 | 删除指定的包版本 |
| 健康检查 | 13 | 检查系统状态 |
| 生成下载链接 | 14 | 获取包的下载地址 |
| 退出 | 0 | 退出程序 |

### 3. 典型使用场景

#### 场景1: 上传新包

```
1. 选择操作: 1
2. 输入包名: my-tool
3. 输入版本: 1.0.0
4. 选择文件: ./my-tool-1.0.0.tar.gz
5. 输入描述: (可选)
6. 输入标签: cli,python (可选)
7. 是否公开: y
8. 等待上传完成
```

#### 场景2: 搜索并下载包

```
1. 选择操作: 4 (搜索包)
2. 输入关键词: cli
3. 查看搜索结果
4. 记下包名和版本
5. 选择操作: 14 (生成下载链接)
6. 输入包名和版本
7. 复制下载链接
8. 使用 curl/wget 下载
```

#### 场景3: 管理已有包

```
1. 选择操作: 6 (查看我的上传)
2. 查看包列表
3. 选择操作: 7 (更新包) 或 8 (删除包)
4. 输入要操作的包名和版本
5. 按提示完成操作
```

### 4. 命令行快捷方式

```bash
# 创建别名
echo 'alias r2="cd /root/self_code/web_backend && uv run python scripts/r2_cli.py"' >> ~/.bashrc
source ~/.bashrc

# 使用
r2  # 直接启动
```

### 5. 环境变量配置

```bash
# 添加到 ~/.bashrc 或 ~/.zshrc
export R2_API_URL="http://localhost:8001"
export R2_USER_ID="00000000-0000-0000-0000-000000000000"
```

## ⚠️ 重要提示

1. **运行位置**: 必须在服务器本地运行（localhost 限制）
2. **API 服务**: 确保 API 服务已启动 (`uv run python run.py`)
3. **文件路径**: 支持相对路径和绝对路径
4. **版本格式**: 必须符合语义化版本 (如: 1.0.0)

## 🆘 常见问题

**Q: 脚本无法连接到 API？**
A: 检查 API 服务是否运行: `curl http://localhost:8001/health`

**Q: 提示 403 Forbidden？**
A: 确保在服务器本地运行，或通过 SSH 连接

**Q: 文件上传失败？**
A: 检查文件路径是否正确，文件是否存在

**Q: 如何退出程序？**
A: 选择菜单选项 0，或按 Ctrl+C
